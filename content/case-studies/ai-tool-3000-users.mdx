## ツール誕生の背景

すべては「自分が使いたいものがなかった」から始まりました。

2,500名規模の組織で総務部長をしていると、
「社内ルールを調べる」という作業が思いのほか多いことに気づきます。
就業規則・福利厚生・申請フロー…これらが分散したドキュメントに埋まっていて、
問い合わせを受けるたびに探し回る日々が続いていました。

**「社内規程をRAGで検索できるツールがあれば」**

これが最初のアイデアでした。

## 開発フロー：Claude Codeとの協業

プログラミング歴ほぼゼロの状態から始めました。
使ったのは**Claude Code**だけです。

### Phase 1：プロトタイプ（1週間）

```
[最初のプロンプト]
Next.js App RouterでPDFをアップロードし、
テキストを抽出してOpenAI APIで検索できる
シンプルなWebアプリを作ってください。
認証不要、ローカルで動けばOKです。
```

Claude Codeが生成したコードをそのまま動かしてみると、
PDFの読み込みから検索UIまで**1時間で動くものができました**。

### Phase 2：品質向上（2〜3週間）

プロトタイプを実際に使ってみると課題が見えてきます。

- 検索精度がイマイチ → Embeddingモデルを変更
- 日本語の句読点でチャンクが分断される → チャンク戦略を改善
- レスポンスが遅い → ストリーミング対応

すべてClause Codeに「〇〇という問題が起きているので改善して」と指示して解決。

### Phase 3：本番デプロイ（1週間）

Vercel + Supabaseの構成でデプロイ。
認証（Supabase Auth）・データ永続化・利用ログの記録まで
Claude Codeが設計から実装まで担当してくれました。

## 3,000人に届くまで

最初は社内の同僚10人に使ってもらいました。
SNSで「こんなもの作った」と投稿したところ、
問い合わせが増え、徐々にユーザーが広がっていきました。

利用者の属性：
- 総務・人事部門の担当者（最多）
- 中小企業の経営者・バックオフィス
- AIツール開発に興味のあるエンジニア

## 収益化の仕組み

ツール自体は**無料**で提供しています。
収益はこのサイトのアフィリエイトと、関連コンテンツから発生しています。

| 収益源 | 概要 |
|-------|------|
| ツール関連アフィリエイト | 使用するAI APIサービスの紹介料 |
| 有料テンプレート販売 | プロンプトセット・GASスクリプト |
| コンサルティング問い合わせ | 法人向けのカスタマイズ相談 |

## 開発コストの実態

| 項目 | コスト |
|------|-------|
| Claude Code（AI開発費） | 月$20〜50 |
| Vercel（ホスティング） | ¥0（無料枠） |
| Supabase（DB） | ¥0（無料枠） |
| ドメイン | ¥1,400/年 |
| **合計** | **月¥3,000〜6,000程度** |

外部発注なら数百万円の開発費になるシステムが、
**月6,000円以下のランニングコスト**で動いています。

## 学んだこと

### 1. 「動くもの」が最強の説明資料

提案書や企画書より、動くプロトタイプの方が圧倒的に伝わります。
AIがある今、「作って見せる」は誰でもできるスキルになりました。

### 2. 実務課題 × AI = 最速の差別化

「AIツールのデパート」になるより、
自分の実務知見とAIを組み合わせた「ニッチな強み」の方が価値があります。

### 3. 収益化は後から考えていい

最初から収益を求めると、ユーザーに届く前に燃え尽きます。
まず「自分が使いたいもの」を作り、ユーザーが集まってから収益設計を考える順序が正解でした。
