## 「顧客が使うアプリ」の重さ

社内ツールと顧客向けアプリでは、品質への要求水準が全く異なります。
社内ツールのバグは「ちょっと不便」で済みますが、
顧客向けアプリのバグは**信頼の失墜と問い合わせ爆増**に直結します。

2,000名規模の顧客が利用するアプリを、少人数で開発・保守する状況で
どうやって品質を担保するか。AIを活用した体制構築の実録です。

## アーキテクチャの概要

```
Next.js (App Router)
  ├── Supabase (DB + Auth)
  ├── Vercel (デプロイ)
  └── GitHub Actions (CI/CD)
       ├── Lint (ESLint)
       ├── 型チェック (tsc)
       ├── ユニットテスト (Jest)
       └── E2Eテスト (Playwright)
```

## Claude Codeとのコードレビュー体制

コード修正時のフローを以下のように標準化しました。

### Step 1：修正前のリスク確認

```
[Claudeへの指示テンプレート]
以下のコードを修正する前に、
潜在的なバグリスクと影響範囲を洗い出してください。

[修正予定のコード]
（コードをここに貼り付け）

確認ポイント：
- エッジケース（空文字・null・大量データ）
- 既存機能への影響
- パフォーマンスへの影響
- セキュリティリスク
```

### Step 2：テストコードの自動生成

```
[Claudeへの指示テンプレート]
以下の関数に対してJestのユニットテストを作成してください。
- 正常系（主要なユースケース）
- 異常系（エラーハンドリング）
- エッジケース（境界値・空値）

[テスト対象の関数]
（関数コードをここに貼り付け）
```

Claude Codeが生成するテストは網羅性が高く、
人間が書くよりエッジケースの漏れが少ないです。

### Step 3：GitHub Actionsでの自動実行

```yaml
# .github/workflows/ci.yml（抜粋）
name: CI
on: [pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install
        run: npm ci
      - name: Type check
        run: npm run tsc
      - name: Test
        run: npm run test -- --coverage
      - name: Lint
        run: npm run lint
```

PRを出した瞬間にこれが走り、全部パスしないとマージできない仕組みです。

## 品質改善の数値

| 指標 | 導入前（3ヶ月平均） | 導入後（3ヶ月平均） |
|------|------------------|------------------|
| 本番バグ報告件数 | 月12件 | 月2〜3件 |

| コードレビュー時間 | 1件あたり60分 | 1件あたり20分 |

**バグ発生率を約80%削減**しながら、開発スピードは逆に上がりました。

## 特に効果的だった施策

### エラーメッセージの改善

「エラーが出ました」という画面を見た顧客からの問い合わせが激減した施策です。
Claude Codeに「このエラー画面を、技術知識のない人が読んで次の行動がわかる文言に改善して」
と指示するだけで、UXを大幅に改善できました。

### 自動ロールバック設定

Vercelのデプロイ設定でLighthouseスコアが閾値を下回ると
自動でロールバックする設定を入れました。
「気づいたら遅くなっていた」という問題が完全になくなりました。

## 「AIにコードを書かせること」のリスク管理

AIが生成したコードを盲目的に使うのは危険です。
以下のルールを設けています。

1. **生成されたコードは必ず自分で読む**（コピペ禁止）
2. **型エラーがある状態でマージしない**
3. **セキュリティに関わる部分はダブルチェック**（Claudeに「このコードのセキュリティリスクを指摘して」と追加確認）

## まとめ

「少人数でも顧客向けアプリを安全に運用できる」は、
AIとCI/CDの組み合わせで現実のものになります。

重要なのは**仕組みを作ること**です。
毎回手動でチェックする文化より、「パスしないとデプロイできない」構造の方が、
長期的な品質維持に圧倒的に効きます。
