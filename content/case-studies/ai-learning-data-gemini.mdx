## きっかけ：「AIに自分を知ってほしい」

ChatGPTやClaudeを使っていて感じる最大の不満は、
「毎回ゼロから自己紹介しなければならない」点です。

私の仕事・思考スタイル・よく使う言い回し…
これらをAIが事前に知っていれば、会話の精度が劇的に上がるはずです。

2年間蓄積したGeminiとの会話履歴（旧Bard含む）が最良の学習素材でした。

## Step 1：Google Takeoutでデータ抽出

Googleアカウントの設定から「データのダウンロード」→「Bard」を選択すると、
会話履歴がJSON形式でダウンロードできます。

私の場合、2年間で**約1,200回の会話**（約180,000トークン相当）がありました。

## Step 2：Claude Codeでデータ整形

ダウンロードしたJSONは生データで、そのままでは使いにくい構造でした。
Claude Codeに整形を依頼しました。

```
[Claudeへの指示]
以下の形式のGemini会話履歴JSONを、
学習データとして使いやすい形式に変換してください。

目標フォーマット：
{
  "conversations": [
    {
      "date": "YYYY-MM-DD",
      "topic": "推定トピック（1行）",
      "my_question": "私の質問・依頼",
      "ai_response": "AIの回答",
      "context": "業務/個人/学習/創作"
    }
  ]
}

追加処理：
- 個人情報（固有名詞・メールアドレス等）をマスク
- 短すぎる会話（3往復未満）は除外
- トピックを自動分類：業務効率化/AI活用/ガジェット/その他
```

Claude Codeが生成したPythonスクリプトを実行すると、
1,200回の会話が**きれいな構造化データ**に変換されました。

## Step 3：分析——自分の思考パターンを可視化

整形したデータをClaude Codeに分析させました。

### 質問パターンの分析結果

| カテゴリ | 割合 | 主なテーマ |
|---------|------|---------|
| 業務効��化 | 38% | 自動化・業務フロー改善 |
| AI活用 | 29% | プロンプト最適化・新ツール調査 |
| 技術学習 | 18% | Next.js・Python・GAS |
| ガジェット | 10% | スペック比較・購入相談 |
| その他 | 5% | 雑談・アイデア出し |

### 発見：自分の「AI活用スタイル」

分析を通じて気づいた自分の特徴：

- **具体的な数値を好む**（「時短できる」より「月20時間削減できる」と聞きたい）
- **実務への応用を常に意識**（理論より実践例を求める傾向）
- **失敗事例に強い関心**（成功例より「何がうまくいかなかったか」を重視）

## Step 4：パーソナライズプロンプトの作成

分析結果を元に、AIとの会話開始時に使う「自己紹介プロンプト」を作りました。

```
【私のプロフィール（AIへの事前情報）】
職業：2,500名規模組織の現役総務部長
専門：AI活用・業務自動化・バックオフィス最適化
思考スタイル：
- 数値による効果検証を重視
- 実務での再現性を最優先
- 失敗事例から学ぶことを好む
よく使う業務：GAS・Python・Next.js・Claude Code
現在の課題：[その日の課題をここに入力]
```

このプロンプトを会話の冒頭に置くと、
AIの回答精度が体感で**大幅に向上**しました。

## プライバシーと倫理の考慮

この実験で特に注意したこと：

1. **個人情報のマスキング**：取引先名・社員名等はすべて除外
2. **ローカル処理を優先**：センシティブなデータは外部APIに送らない
3. **データの保管場所**：暗号化したローカルストレージのみ

整形・分析することで、自分が何に時間を使い、何に迷っているかが見えてきます。
