## はじめに：WPの「限界」を感じた瞬間

このサイト（ainaviz.com）は、もともとWordPressで運用していました。
テーマのカスタマイズ、プラグインの更新管理、セキュリティパッチ対応…
「情報を発信するためのブログ」なのに、**運用そのものが業務**になっていました。

決定打はPageSpeed Insightsのスコアでした。モバイルで**38点**。
「AIで最適化したサイト」と名乗るには、あまりにも恥ずかしい数値です。

## 移行の判断

移行先の条件は以下の通り定めました。

- **静的生成（SSG）** でPageSpeed 90点以上を目指す
- **Markdownでコンテンツ管理** できる（DBレス）
- **Vercelにデプロイ** してホスティングコストを削減
- **Claude Codeを使って** 自分でコードが書けること

Next.js App Routerが最適解でした。

## Claude Codeとの協業フロー

移行作業の大部分は**Claude Codeに指示を出しながら実装**しました。
具体的には以下のような流れです。

```
1. 既存WPテーマのHTML/CSSをClaude Codeに渡す
2. 「これをNext.js Tailwind CSSで再現して」と指示
3. 生成されたコードをローカルで確認・フィードバック
4. 繰り返し改善
```

特にMDXによるコンテンツ管理の仕組み（`gray-matter` + `next-mdx-remote`）は
Claude Codeが一発で動くコードを生成してくれました。

## 移行前後の比較

| 指標 | WordPress | Next.js (現在) |
|------|-----------|----------------|
| PageSpeed (モバイル) | 38点 | 97点 |
| PageSpeed (PC) | 62点 | 100点 |
| 月次ホスティング費用 | ¥2,200/月 | ¥0（Vercel無料枠） |
| プラグイン管理 | 毎月3〜5時間 | 不要 |
| コンテンツ追加 | WP管理画面 | VSCode + MDX |
| デプロイ | FTPで手動 | git pushで自動 |

## ハマったポイントと解決策

### 1. MDXのコードハイライト

`rehype-prism-plus` の導入でハマりました。
Claude Codeに「prismjsのtheme-tomorrowをNext.jsで適用する方法」と聞いたところ、
RSCとCSS importの順序まで含めて正確に教えてもらえました。

### 2. 画像の最適化

WPで使っていた画像をそのまま持ってきたところ、LCPが悪化。
`next/image` コンポーネントへの置き換えとWebP変換を指示すると、
Claude Codeがすべての画像タグを一括置換してくれました。

### 3. OGP設定

App RouterのMetadata APIの書き方が新しく、WP時代のプラグインとは別物。
`generateMetadata` の書き方をClaude Codeが正確に生成してくれたので迷わず実装できました。

## 結果

- **月¥2,200の費用が¥0**に（年間¥26,400の削減）
- **PageSpeedモバイル 38点 → 97点**
- コンテンツ追加が「WP管理画面を開く」から「MDXファイルを作る」に変わり、むしろ速くなった

「AIで最適化したサイト」という実績を、このサイト自体で証明できています。

## まとめ

WP→Next.js移行は、プログラミング経験がなくても**Claude Codeがあれば実現可能**です。
ポイントは「何を作るか」ではなく「何を改善したいか」を明確にして指示を出すこと。
